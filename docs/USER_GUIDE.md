## 应用概述

GitLab Auto Deploy 是一个桌面应用程序，用于自动化 GitLab Web 项目的部署流程。它能够：

- 监听指定 GitLab 项目的指定分支
- 当分支有更新（MR 合并）时自动触发部署
- 自动执行：克隆代码 → 构建 → 上传到服务器
- 支持后台守护模式，无需手动干预

**支持平台**：macOS、Windows

## 快速开始

### 第一步：配置 GitLab 连接

1. 点击左侧菜单 **系统设置** → **GitLab 连接**
2. 点击 **添加连接** 按钮
3. 填写以下信息：
   - **连接名称**：自定义名称，如"公司 GitLab"
   - **API URL**：GitLab API 地址
     - GitLab.com 用户：`https://gitlab.com`
     - 私有部署：`https://your-gitlab.com`
   - **Access Token**：GitLab 个人访问令牌
     - 获取方式：GitLab → 用户设置 → Access Tokens → 新建令牌
     - 所需权限：`read_api`、`read_repository`

4. 点击 **测试连接** 验证配置
5. 点击 **保存**

### 第二步：配置服务器

1. 点击左侧菜单 **系统设置** → **服务器管理**
2. 点击 **添加服务器** 按钮
3. 填写以下信息：
   - **服务器名称**：自定义名称，如"生产服务器"
   - **主机地址**：服务器 IP 或域名
   - **SSH 端口**：默认 22
   - **用户名**：SSH 登录用户名
   - **部署路径**：项目部署目录，如 `/var/www/html/project-name`
   - **健康检查 URL**（可选）：用于验证部署是否成功

4. 点击 **测试连接** 验证 SSH 连接
5. 点击 **保存**

### 第三步：添加项目

1. 点击左侧菜单 **项目管理**
2. 点击 **添加项目** 按钮
3. 填写项目信息：
   - **项目名称**：自定义显示名称
   - **GitLab 路径**：项目路径，格式：`group/project` 或 `username/project`
   - **分支**：监听的分支名称，默认 `main`
   - **部署路径**：服务器上的部署目录
   - **构建命令**：项目构建命令，如 `npm run build`
   - **输出目录**：构建输出目录，如 `dist`

4. 点击 **保存**

### 第四步：启用自动部署

1. 点击左侧菜单 **系统设置** → **常规设置**
2. 开启 **启用守护进程** 开关
3. 设置 **轮询间隔**（检查 GitLab 更新的频率）
4. 点击 **保存设置**

## 功能模块

### 项目管理

项目管理是应用的核心模块，用于配置需要自动部署的项目。

#### 项目列表

显示所有已配置的项目，包含以下信息：
- 项目名称
- GitLab 路径
- 监听分支
- 部署路径
- 创建时间

#### 操作说明

| 操作 | 说明 |
|------|------|
| 添加项目 | 创建新的部署项目配置 |
| 详情 | 查看项目详细信息和部署历史 |
| 编辑 | 修改项目配置 |
| 删除 | 删除项目（已有部署记录的项目需确认） |

#### 项目详情页

点击项目的 **详情** 按钮进入详情页，可查看：
- 项目完整配置信息
- 部署历史记录
- 手动触发部署按钮

---

### 部署队列

部署队列显示所有部署任务的状态和进度。

#### 状态说明

| 状态 | 说明 | 颜色 |
|------|------|------|
| 等待中 | 任务已创建，等待执行 | 橙色 |
| 执行中 | 正在执行部署流程 | 蓝色 |
| 成功 | 部署完成 | 绿色 |
| 失败 | 部署过程中出现错误 | 红色 |
| 已取消 | 用户手动取消 | 灰色 |

#### 筛选标签

- **全部**：显示所有部署记录
- **运行中**：仅显示正在执行的任务
- **已完成**：显示成功、失败、已取消的任务

#### 操作说明

| 操作 | 说明 |
|------|------|
| 日志 | 查看部署详细日志 |
| 取消 | 取消正在执行的任务 |

### 系统设置

系统设置包含三个子模块：

#### 1. 常规设置

**守护进程配置**

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 启用守护进程 | 开启后自动监听项目更新并触发部署 | 关闭 |
| 轮询间隔 | 检查 GitLab 更新的频率 | 1 分钟 |

**通知设置**

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 部署成功通知 | 部署完成时发送通知 | 开启 |
| 部署失败通知 | 部署失败时发送通知 | 开启 |

#### 2. GitLab 连接

管理 GitLab 服务器连接配置。

**配置项说明**

| 字段 | 必填 | 说明 |
|------|------|------|
| 连接名称 | 是 | 自定义名称，便于识别 |
| API URL | 是 | GitLab API 地址 |
| Access Token | 是 | GitLab 个人访问令牌 |

**获取 Access Token**

1. 登录 GitLab
2. 点击右上角头像 → **Preferences** (偏好设置)
3. 左侧菜单选择 **Access Tokens**
4. 点击 **Add new token**
5. 填写：
   - Name: 令牌名称
   - Expiration: 过期时间
   - Scopes: 勾选 `read_api` 和 `read_repository`
6. 点击 **Create personal access token**
7. 复制生成的令牌（只显示一次）

#### 3. 服务器管理

管理目标部署服务器。

**配置项说明**

| 字段 | 必填 | 说明 |
|------|------|------|
| 服务器名称 | 是 | 自定义名称 |
| 主机地址 | 是 | IP 或域名 |
| SSH 端口 | 否 | 默认 22 |
| 用户名 | 是 | SSH 登录用户 |
| 部署路径 | 否 | 默认部署目录 |
| 健康检查 URL | 否 | 部署后验证地址 |

## 工作流程

### 自动部署流程

```
┌─────────────────────────────────────────────────────────────┐
│                      自动部署流程                            │
└─────────────────────────────────────────────────────────────┘

1. 监听阶段（守护进程模式）
   └─→ 轮询 GitLab API 检查项目分支更新

2. 触发阶段
   └─→ 检测到 MR 合并或推送事件
   └─→ 创建部署任务，状态：等待中

3. 执行阶段
   ├─→ 克隆/拉取代码
   ├─→ 执行构建命令 (npm run build)
   ├─→ 打包输出目录
   ├─→ SSH 连接服务器
   ├─→ SFTP 上传文件
   └─→ 解压并替换旧版本

4. 完成阶段
   ├─→ 成功：状态更新为"成功"，发送通知
   └─→ 失败：状态更新为"失败"，发送通知，保留日志
```

### 手动部署流程

1. 进入 **项目管理**
2. 点击目标项目的 **详情**
3. 点击 **开始部署** 按钮
4. 在 **部署队列** 查看进度

## 常见问题

### Q1: 连接 GitLab 失败？

**可能原因**：
- API URL 格式错误
- Access Token 无效或过期
- 网络无法访问 GitLab

**解决方案**：
1. 确认 API URL 正确（包含 `https://` 前缀）
2. 在 GitLab 中重新生成 Access Token
3. 检查网络连接和代理设置

### Q2: SSH 连接服务器失败？

**可能原因**：
- 服务器地址或端口错误
- 用户名错误
- SSH 密钥未配置
- 防火墙阻止连接

**解决方案**：
1. 确认服务器 IP 和端口可访问
2. 验证用户名是否正确
3. 确保 SSH 密钥已添加到服务器的 `~/.ssh/authorized_keys`
4. 检查服务器防火墙规则

### Q3: 部署失败？

**可能原因**：
- 构建命令错误
- 输出目录不存在
- 服务器磁盘空间不足
- 权限不足

**解决方案**：
1. 检查构建命令是否正确
2. 确认输出目录路径正确
3. 检查服务器磁盘空间
4. 确保用户对部署目录有写入权限

### Q4: 守护进程不工作？

**可能原因**：
- 守护进程未启用
- 轮询间隔过长
- GitLab 连接失败

**解决方案**：
1. 检查 **常规设置** 中守护进程是否启用
2. 确认 GitLab 连接正常
3. 查看应用日志获取详细错误信息

### Q5: 如何查看部署日志？

1. 进入 **部署队列**
2. 找到对应的部署任务
3. 点击 **日志** 按钮查看详情
